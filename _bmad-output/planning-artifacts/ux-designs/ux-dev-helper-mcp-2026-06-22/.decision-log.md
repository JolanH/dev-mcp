## Inherited facts (from inputs, not re-decided here)
- Read-only dashboard, single human-facing surface; localhost single-operator. (FR-8/9/10)
- Primary metric SM-2: state every running agent's work in < ~10s at a glance.
- Board grouped BY TASK; each task = description + one status (running/blocked/done) + 1+ per-repo worktrees (repo, branch, path).
- Live via ~1–2s polling of `/state`, ≤3s freshness (bounded ≤15 repos).
- Minimal server-rendered HTML + vanilla-JS poller; no SPA/build tooling; no external assets (localhost, strict no-egress).
- Pinned guardrails to honor: subordinate `generated_at` (greys/ambers when stale); orphaned-annotation section demoted/below-fold, self-explaining; stable render (DOM changes only on state change); labeled git-unavailable degradation (never blank); per-repo degrade.

## Revisions (Party Mode 2026-06-22, post-finalize)
- 2026-06-22 — **Done is a folded count, not a 4th column.** Board = **3 active columns** (Running | Blocked | Review); Done collapses to a `✓ N done` disclosure below the board, unfoldable to its done cards (dimmed). Rationale (Sally + John): Done needs no operator action and grows unbounded across a day; a permanent column dies by accretion and competes with the actionable states. Summary bar still shows the done count.
- 2026-06-22 — **Emphasis moves Running → Blocked.** Blocked is the only "alarm" state (lifted card + amber glow): an idle agent is the most urgent thing. Running becomes the calm, flat baseline (it needs nothing from the operator). Urgency gradient: blocked → review → running → done. Supersedes the earlier "activity-forward / lift running" choice.
- 2026-06-22 — **Per-card status glyph (●/▲/◆/✓) is on every card**, not just column headers — the non-color channel must travel with each card (Sally: bar color alone is unsafe for deuteranopia given teal/blue proximity).
- 2026-06-22 — **Drop "merge" from review wording.** Review = "awaiting your review" / badge "awaiting review" — never "ready to merge." Rationale (John): merge-back is explicitly out of scope; "merge" language promises a button that doesn't exist. The dashboard's job ends at "this needs your eyes."
- 2026-06-22 — **Overflow contract.** Active columns are bounded by agent count (2–4 typical) and fit without scroll; a column that overflows scrolls **within the column**; the board never scrolls horizontally (narrow widths wrap, mobile out of scope).
- 2026-06-22 — **Stale threshold pinned:** the freshness stamp shows the stale treatment when `generated_at` is older than **2 × poll-interval** (~3–4s). Makes UX-DR6/7 machine-checkable.
- 2026-06-22 — **Poller = diff-and-patch, not wholesale innerHTML replace.** Keyed by `task_id`, per-task content hash; identical snapshot → zero DOM writes; status change reparents the existing node. Required so unfolded Done/orphan sections and scroll survive a poll. UX-DR5 testable via a pure `diff(prev,next)→patches` function (`diff(x,x)===[]`) under `node --test` — no Playwright, no jsdom.
- 2026-06-22 — **No-Playwright test strategy (Murat):** UX-DRs tested via HTML-output assertions (pytest) + static CSS/JS lint (grep for transition/animation/external assets) + a pure WCAG-contrast math check over the token pairs. Adds ~1 HTML parser + a tiny contrast fn + `node --test` for the diff fn.
- 2026-06-22 — **Empty-state copy:** empty Blocked column reads "Nothing needs you" (a feature, not blank); empty board reads "No active tasks — create one with create_task"; empty Review/Running show header + "0".

## Decisions
- 2026-06-22 — **Layout = status columns (kanban): Running | Blocked | Done.** Task cards flow into their status column; per-repo worktrees nest inside the card. Rationale: position encodes status, serving the <10s glance (SM-2); "what's blocked" is found spatially, not by reading badges.
- 2026-06-22 — **Theme = dark.** Rationale: monitor tab left open beside the editor; status colors pop against dark; terminal/dev-tool affinity.
- 2026-06-22 — **Glance emphasis = activity-forward.** Running tasks carry a subtle liveness cue (e.g. soft pulse); done is dimmed/recedes; blocked stays distinct (amber). Note: combined with kanban columns, position handles "where is blocked" while the liveness cue handles "what is actively moving" — complementary, not conflicting.
- 2026-06-22 — **Discovery depth = coaching:** render HTML visual directions before committing the spines.
- 2026-06-22 — [ASSUMPTION] Single screen, web/desktop browser tab on localhost; no mobile surface (single-operator at their workstation). To confirm.
- 2026-06-22 — **Visual direction = "Modern console, compact" (merge of Direction B look-and-feel + Direction C density).** Keep from B: dark surface, rounded cards with soft elevation, teal running accent + faint glow, top summary-count pill bar, system sans-serif, blocked badge, demoted `<details>` orphan section. Take from C: tighter type scale (~11.5–13px), compact card padding (~9px), reduced gaps/margins, worktrees as small monospace sub-lines — built to hold ~4–8 tasks without scrolling. Rationale: B reads as a calm product surface (fits SM-1 "trust it, stays out of the way") while C's density keeps the whole board in one glance (SM-2).
- 2026-06-22 — **No motion / no pulsing dots.** All status dots and the freshness dot are static. Rationale: the dashboard is a tab left open all day; constant animation is fatiguing and a poor fit for `prefers-reduced-motion`. Running emphasis is carried by color/accent + glow instead.
- 2026-06-22 — **Status-colored left-edge bar on every card (from Direction C).** A 3px bar in the status color (teal running / amber blocked / muted-grey done), clipped to the card's rounded corners. Rationale: a second, position-anchored status cue beyond the column and the dot — strengthens the glance and aids non-color perception alongside the column header glyphs (●/▲/✓).
- 2026-06-22 — [NOTE FOR UX] Status is triple-encoded (column position + left bar color + dot/glyph). Intentional redundancy for glanceability + colorblind safety; keep the column-header glyph as the non-color channel. Confirm contrast at Finalize (accessibility floor).
- 2026-06-22 — **Canonical status set expands 3 → 4: `{running, blocked, review, done}`.** `blocked` = agent awaiting user **input** (stuck mid-work); `review` = agent **finished**, awaiting user review/merge (happy-path gate before `done`). `review` is an active/non-`done` state (counts toward one-active-task-per-slug); `done` remains terminal (reviewed/merged/closed). **CROSS-CUTTING — not UX-only:** changes FR-6, the SQLite `status CHECK`, the `update_task` status enum, and the dashboard column set. To be propagated to PRD/architecture/addendum/epics when the epics workflow resumes (treat as a second amendment).
- 2026-06-22 — **Board is now 4 columns: Running | Blocked | Review | Done** (lifecycle order; the two "needs-me" states sit between active and closed). Colors: running teal `#39d0a8`, blocked amber `#e3a34a`, review blue `#6cb6ff`, done grey `#7d8590` (dimmed). Column glyphs (non-color channel): ● ▲ ◆ ✓. Badges carry the reason ("needs input" / "ready to merge").
