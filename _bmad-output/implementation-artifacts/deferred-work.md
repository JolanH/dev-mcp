# Deferred Work

## Deferred from: code review of story-1-1 (2026-06-22)

- **TOCTOU port race** [src/dev_helper_mcp/server.py:18-44] — `find_free_port` binds+closes a probe socket, then `uvicorn.run` re-binds the same port by number. Between the probe close and the uvicorn bind another process can grab the port, crashing with an unhandled `OSError` (no retry/fallback). `SO_REUSEADDR` on the probe widens the gap. Robust fix: hand the already-bound socket to uvicorn (`uvicorn.Server` with a passed socket) instead of re-binding by number. Deferred to Story 3.1 (single-instance protection / lockfile); low real-world impact for a localhost single-user tool.
- **`Mount("/", app=mcp_app)` shadows future routes** [src/dev_helper_mcp/server_factory.py:75] — the MCP sub-app is mounted at `/` and owns the entire URL space, so any dashboard/sibling route later added to the parent Starlette app will be unreachable. This wiring is the mandated 307-fix for Story 1.1, but Epic 2's dashboard (`/state` + board at `/`) must revisit it — e.g. register dashboard routes inside the MCP app, mount MCP at a sub-path, or order routes so the dashboard wins. Deferred to Epic 2.

## Deferred from: code review of story-1-2 (2026-06-22)

- **Store multi-statement atomicity** [src/dev_helper_mcp/store.py] — `add_task`/`add_worktree` each `commit()` individually, so `create_task`'s `add_task` + N×`add_worktree` is not one transaction; a crash mid-sequence leaves an orphan task row, and a duplicate `task_id`/bad status surfaces as a raw `aiosqlite.IntegrityError` escaping core. Story 1.3 (`create_task`) must add an explicit transaction boundary (rows written LAST, single SQLite transaction, all-or-nothing per architecture §create_task atomicity) and map `IntegrityError` → typed `ActiveTaskConflict`/`BranchExists`. Out of Story 1.2's substrate scope (the primitives are correct in isolation). Deferred to Story 1.3.
- **Process-group kill for git children** [src/dev_helper_mcp/git/runner.py:_exec] — timeout `proc.kill()` SIGKILLs only the direct `git` process; `worktree add` can spawn hooks / `git gc --auto` / credential helpers that survive. Robust fix: `start_new_session=True` on spawn + `os.killpg` on timeout so the whole subtree is reaped. Low impact (mutation timeout is 120s, rarely hit; localhost single-user). Deferred, hardening.
- **HOME-unset / relative `XDG_STATE_HOME`** [src/dev_helper_mcp/config.py:state_dir] — with `HOME` unset and no `XDG_STATE_HOME`, `os.path.expanduser("~")` can return the literal `~`, and a relative `XDG_STATE_HOME` is used verbatim, so `state_dir()` resolves to a CWD-relative path (DB written somewhere unexpected, contradicting the "machine-global, never under cwd" promise). Fix: absolutize the result and fall back to a sane default when `~` cannot expand. Rare under normal `uv tool install` use (HOME set). Deferred, env-edge.
- **RepoLockRegistry unbounded growth** [src/dev_helper_mcp/git/repo_lock.py] — per-repo locks are minted lazily and never evicted, a slow memory leak for a long-lived server touching many distinct repos. Consider weak-ref / LRU eviction of idle locks. Negligible at v1 single-user scale. Deferred.

## Deferred from: code review of story-1-3 (2026-06-22)

- **Post-preflight partial state / no compensation** [src/dev_helper_mcp/core/tasks.py:116-133] — once provisioning starts, a `worktree add` failure on repo N (race, disk error) leaves repos 1..N-1's worktrees + `agent/<slug>` branches orphaned on disk with no DB row and no teardown; the failure also surfaces as a generic `Internal` rather than typed `BranchExists`/`WorktreePathInUse` even when the cause is a post-preflight collision. Story 1.3 is happy-path + preflight only — full reverse-order compensation, `RollbackIncomplete`, and the `RepoMutator` fault-injection seam are explicitly Story 1.4. Deferred to Story 1.4.
- **Per-repo mutex is process-local** [src/dev_helper_mcp/core/tasks.py:63-70] — the `RepoLockRegistry` serializes only coroutines within one process; two server processes pointing at the same repos/state would not serialize, and even within one process two same-slug creates over *disjoint* repo sets share no lock, so both can pass the `status != 'done'` gate and provision worktrees before the persist-time `ActiveTaskConflict` rejects the loser (orphaning its worktrees). Mitigated by the machine-global single-instance lockfile (Story 3.1, which enforces one server process) + post-failure rollback (Story 1.4). Deferred.

## Deferred from: code review of story-1-4 (2026-06-23)

- **`except DevHelperError` doesn't catch `CancelledError`/`BaseException`** [src/dev_helper_mcp/core/tasks.py:143] — the provisioning try/except only triggers compensation on a `DevHelperError`. A task cancellation (server shutdown, client disconnect) or any non-typed exception raised mid-`mutator.add` propagates without running the reverse-order teardown, leaking already-provisioned worktrees + `agent/<slug>` branches. In production `add`/`remove` only raise `DevHelperError` (verified: `run_git` raises only `GitTimeout`/`NotAGitRepo`/`Internal`, all `DevHelperError`), so the live exposure is `asyncio.CancelledError`. Adjacent to the explicit v1 crash-safety non-goal; a correct fix must `asyncio.shield` the teardown so cancellation doesn't abort it mid-rollback. Deferred — hardening, non-trivial.
- **`_classify_add_failure` is locale-fragile** [src/dev_helper_mcp/core/mutator.py:528-546 / src/dev_helper_mcp/git/runner.py:132] — typed classification of a non-zero `git worktree add` matches English stderr substrings (`"a branch named"`, `"is already checked out"`, `"already exists"`). The runner env (runner.py:132) strips `GIT_*` context vars and pins `GIT_TERMINAL_PROMPT`/`GIT_OPTIONAL_LOCKS` but does NOT pin `LC_ALL=C`/`LANG=C`, so under a non-English system locale git's localized stderr falls through to the generic `Internal`. Degrades safely (compensation still runs; story sanctions `Internal` as the best-effort fallback), but an agent branching on the stable typed code mis-handles the collision. Fix: pin `LC_ALL=C` in the runner env (also hardens all future porcelain parsing) — edits `runner.py`, which Story 1.4 holds unchanged, hence deferred.
- **`remove`'s `branch -D` force-deletes unguarded at the reusable seam boundary** [src/dev_helper_mcp/core/mutator.py:600] — `GitRepoMutator.remove` unconditionally runs `git branch -D <branch>`. Safe in 1.4 (the branch was just created by this very call, so it is brand-new + unmerged and `-D` is correct), but the module docstring explicitly anticipates Story 1.5 extending/reusing this seam for the user-facing `remove_worktree` tool. A future caller invoking `remove` on a pre-existing branch would silently force-delete unmerged work. Guard the destructive `-D` at the API boundary (or rename to a teardown-only method) before Story 1.5 reuses it. Deferred to Story 1.5.

## Deferred from: code review of story-1-5 (2026-06-23)

- **Vanished/renamed repo: orphaned link not cleanable** [src/dev_helper_mcp/core/worktrees.py] — if a `task_worktree` link's repo dir is deleted/renamed, `remove_worktree`'s `git worktree remove` returns a non-zero ("cannot change to dir") that matches none of `_is_not_a_worktree`/`_is_locked`/`_is_dirty`, so it raises generic `Internal` and the link can never be cleaned via this path. The story arguably wants vanished-repo links removable, but link reconciliation / startup sweep is an explicit v1 non-goal (scope fence). Deferred — reconciliation.
- **Moved worktree path not reconciled** [src/dev_helper_mcp/core/worktrees.py] — `list_worktrees` computes `orphaned` purely from `(repo_path, branch)` membership in the live porcelain (per spec). A worktree relocated with `git worktree move` (branch present at a different path) reads `orphaned:false` while the reported `worktree_path` is stale. `add_worktree`/move is deferred project-wide (AR-13). Deferred.
- **Detached-HEAD agent worktree reads orphaned** [src/dev_helper_mcp/git/porcelain.py] — a worktree in detached-HEAD state parses to `branch=None`, is excluded from the live branch set, and its stored link flags `orphaned:true` even though the worktree exists on disk. False positive; nothing reconciles by path. Low impact (agent worktrees are created on `agent/<slug>` and don't normally detach). Deferred — heuristic limitation.
- **Cross-repo same-task concurrent close not serialized** [src/dev_helper_mcp/core/worktrees.py] — the per-repo mutex serializes same-repo mutations only. Two concurrent `remove_worktree` calls for the *same task* in *different* repos hold different locks; the shared `task` row's `count_worktrees`/`delete_task` and the `task_closed` flag are not serialized across them. `delete_task` is idempotent (no crash/corruption) and `task_closed` is cosmetic. Deferred — would need a task-level guard; cosmetic at v1 scale.
- **AC2 "or by path" identifier not implemented** [src/dev_helper_mcp/core/worktrees.py] — `remove_worktree` resolves only by `(task_id, repo)`; the optional path-based identifier (AC2 "or by path") is not wired, though the Task 4 subtask is checked. The story marks the path identifier secondary; the binding ACs use `task_id`+`repo`. Deferred — optional, revisit if a path-keyed remove is needed.
- **LockedWorktree classifier untested** [tests/test_worktrees.py] — the `is locked` → `LockedWorktree` branch in `remove_worktree` has no test (no `git worktree lock` scenario in the suite), so that classifier is unverified. Additive, low-risk. Deferred — add coverage with a `git worktree lock` tmp-repo test.
- **Concurrency test relies on a fixed sleep** [tests/test_concurrency.py] — `test_remove_worktree_is_serialized_by_repo_mutex` uses `asyncio.sleep(0.05)` to let the contender block, and only proves `remove_worktree` blocks at `lock.acquire()` — not that the lock is held across the entire git+persist critical section. A regression that released the lock early between git and the store write would still pass. Matches the existing suite's timing pattern. Deferred — test hardening.
- **AC1 `-z` literal deviation** [src/dev_helper_mcp/git/porcelain.py] — epics AC1 quotes `git worktree list --porcelain -z`, but the dev git (2.34.1) errors on `worktree list -z` (added in git 2.36), so the code invokes `--porcelain` without `-z`. AC1's binding "live, not a cache" intent is fully met and the parser is delimiter-agnostic (NUL-form ready). Operator-approved. Deferred — reconcile the epics AC text or pin git ≥ 2.36 (documentation follow-up).

## Deferred from: code review of 2-2-in-memory-cache-and-background-refresher (2026-06-25)

- **`repo_path` spelling/normalization in the cache fan-out** [src/dev_helper_mcp/cache.py] — `refresh()` derives the distinct-repo set and keys `_last_listings`/`git_listings` on the raw stored `repo_path` string.
