# Deferred Work

## Deferred from: code review of story-1-1 (2026-06-22)

- **TOCTOU port race** [src/dev_helper_mcp/server.py:18-44] — `find_free_port` binds+closes a probe socket, then `uvicorn.run` re-binds the same port by number. Between the probe close and the uvicorn bind another process can grab the port, crashing with an unhandled `OSError` (no retry/fallback). `SO_REUSEADDR` on the probe widens the gap. Robust fix: hand the already-bound socket to uvicorn (`uvicorn.Server` with a passed socket) instead of re-binding by number. Deferred to Story 3.1 (single-instance protection / lockfile); low real-world impact for a localhost single-user tool.
- **`Mount("/", app=mcp_app)` shadows future routes** [src/dev_helper_mcp/server_factory.py:75] — the MCP sub-app is mounted at `/` and owns the entire URL space, so any dashboard/sibling route later added to the parent Starlette app will be unreachable. This wiring is the mandated 307-fix for Story 1.1, but Epic 2's dashboard (`/state` + board at `/`) must revisit it — e.g. register dashboard routes inside the MCP app, mount MCP at a sub-path, or order routes so the dashboard wins. Deferred to Epic 2.

## Deferred from: code review of story-1-2 (2026-06-22)

- **Store multi-statement atomicity** [src/dev_helper_mcp/store.py] — `add_task`/`add_worktree` each `commit()` individually, so `create_task`'s `add_task` + N×`add_worktree` is not one transaction; a crash mid-sequence leaves an orphan task row, and a duplicate `task_id`/bad status surfaces as a raw `aiosqlite.IntegrityError` escaping core. Story 1.3 (`create_task`) must add an explicit transaction boundary (rows written LAST, single SQLite transaction, all-or-nothing per architecture §create_task atomicity) and map `IntegrityError` → typed `ActiveTaskConflict`/`BranchExists`. Out of Story 1.2's substrate scope (the primitives are correct in isolation). Deferred to Story 1.3.
- **Process-group kill for git children** [src/dev_helper_mcp/git/runner.py:_exec] — timeout `proc.kill()` SIGKILLs only the direct `git` process; `worktree add` can spawn hooks / `git gc --auto` / credential helpers that survive. Robust fix: `start_new_session=True` on spawn + `os.killpg` on timeout so the whole subtree is reaped. Low impact (mutation timeout is 120s, rarely hit; localhost single-user). Deferred, hardening.
- **HOME-unset / relative `XDG_STATE_HOME`** [src/dev_helper_mcp/config.py:state_dir] — with `HOME` unset and no `XDG_STATE_HOME`, `os.path.expanduser("~")` can return the literal `~`, and a relative `XDG_STATE_HOME` is used verbatim, so `state_dir()` resolves to a CWD-relative path (DB written somewhere unexpected, contradicting the "machine-global, never under cwd" promise). Fix: absolutize the result and fall back to a sane default when `~` cannot expand. Rare under normal `uv tool install` use (HOME set). Deferred, env-edge.
- **RepoLockRegistry unbounded growth** [src/dev_helper_mcp/git/repo_lock.py] — per-repo locks are minted lazily and never evicted, a slow memory leak for a long-lived server touching many distinct repos. Consider weak-ref / LRU eviction of idle locks. Negligible at v1 single-user scale. Deferred.
